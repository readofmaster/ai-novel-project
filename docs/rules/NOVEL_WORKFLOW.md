# AIチーム小説作成規約

## チーム役割
- **Writer (作家担当):** ドラフトの執筆、描写、対話、物語の展開を担当。
- **Editor (編集担当):** 構造、テンポ、スタイルの一貫性、プロットの整合性、論理的矛盾のチェックを担当。
- **Reader (読者担当):** エンゲージメント、読みやすさ、キャラクターの好感度、テーマのインパクトに関する客観的なフィードバックを担当。

## 作品の方向性・ルール
- **トーン**: ライト、ポップ、明るい雰囲気。
- **ジャンル・要素**: コメディ、お笑い要素を適宜取り入れる。
- **時代・舞台**: 現代または未来。
- **ボリューム**: 
    - 1回あたりの執筆量: 約800文字（1〜2分で読める量）。
    - 全体目標: 1万文字程度。

## 小説作成ワークフロー
1. **コンセプト・プロット作成:** 全チームメンバーで物語の核となるテーマやプロットを策定する（1万文字の全体構成を作る）。
2. **執筆:** Writerがシーンや章単位で800文字程度の執筆を行う。
3. **編集・校閲:** Editorが内容をチェックし、改善案を提示する。
4. **読者フィードバック:** Readerが視点を共有し、読者の心理や反応を分析する。
5. **改稿:** WriterがEditorとReaderのフィードバックに基づいて改稿を行う。
6. **本文の統合:** 改稿完了後、Writerがデイリーレポートから小説本文を抽出し、対応する章ファイル（`docs/novel/`配下）へ追記・統合する。
7. **完成:** 全チームメンバーが合意し、最終稿を確定する。
